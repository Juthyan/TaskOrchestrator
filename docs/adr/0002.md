# ADR-0002 — CancellationToken in ITaskRepository

## Status
Accepted

## Context
The system must be able to cancel an operation I/O if 
the client disconnect himself instead of pursue the work.

## Decision
In the different call in `ITaskRepository` we put the token with a defaukt value.

## Reasons
- If the client disconnets the task must no continue
- The = default means by default no cancelation. Without it, the system would continue I/O work for disconnected clients, wasting resources
- It is a .NET convention — any async I/O method should accept a CancellationToken
## Consequences
- By now each call using the repository must have the cancelation token
- Callers like `EnqueueTaskCommandHandler` pass their own token down the chain
