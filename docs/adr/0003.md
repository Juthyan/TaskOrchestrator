# ADR-0003 — ConcurrentDictionary

## Status
Accepted

## Context
The system will have multiple workers using the repository for I/O operations
These operations are async then a simple dictionnary cant do it because of the multithreading.

## Decision
Instead of using a dictionnary we use the concurrentDictionnary.

## Reasons
- Multiple I/O operation a the same time coming from different workers.
- Multiple thread wrting in the same disctionnary can cause corrupted datas. (race condition)

## Consequences
- Using ConcurrentDictionnary instead
- ConcurrentDictionnary is thread-safe one writing at a time
