# ADR-0004 — 4 parallel workers

## Status
Accepted

## Context
The system must have 4 workers max working in paralel. But it means 
that we cant pass the workers by the service. The DI wont be able 
to controll them. And this is less maintanable.

## Decision
We use an objetc TaskWorker having for workers inside of it.
Only TaskWorker is passed to the DI.

## Reasons
- One instance TaskWorkers to control
- More maintanable
- the function WhenAll wait for the workers to stop properly
- if a worker crash WhenAll can detect it 

## Consequences
- DI dont have to deal with the mutiltiple
- Easy to increase the number of workers
