# ADR-0005 — dual channel priority

## Status
Accepted

## Context
The system having 4 workers working simultaneously accessing
one and unique queue can cause problem like race condition.
Futhermore just having one queue increase the complexity to
deal with the notion of priority.

## Decision
We implement 2 queues. One high priority the other one low.

## Reasons
- unique queue can cause problem like race condition
- More maintanable
- Priority queue is not thread-safe

## Consequences
- the worker treat the high priority first
- If the high priority queue is empty then the worker take the low one
