# ADR-0007 — TaskChannels

## Status
Accepted

## Context
To avoid to deal with multiple injection of queue in builder 
we created one object to wrap them.
Having a multiple queue injected cause a lot of problem for the DI
and to maintam them
TaskChannels needed to be accessible by both Application (handlers) 
and Infrastructure (worker). Infrastructure can reference Application, 
but not the other way around.

## Decision
We wrap on one objetc the two queue and 
 we place it in Application layer to respect the dependency direction.

## Reasons
- More maintanable
- More manageable

## Consequences
- we call TaskChannels to accesed to the queue
