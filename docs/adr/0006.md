# ADR-0006 — distinction between the automatic retry and the manual one

## Status
Accepted

## Context
A reflexion has been made about the old retry system.
At first we only have the manual one but it does not 
make sens. Because a task is deal by the channel. 
the manuel one is only when 3 attemps has been reach

## Decision
We implement 2 ways to do a retry. Manuel and automatic

## Reasons
- No human inntervention in the worker process
- Possibility to restart a task in some conditions

## Consequences
- the worker now deal with the retry.
- When the 3 attemps has been reach. the task is in fail state
- Someone can decide after the 3 attemps to relaunch
- When a task is restarded then the attemps is going back at 0
